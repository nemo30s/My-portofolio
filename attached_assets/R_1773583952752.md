## Project Overview
This project implements Deep Reinforcement Learning (DRL) using Stable Baselines3 on the **MountainCar-v0** environment. It includes:
1. **Baseline**: Training with default reward function
2. **Custom Reward**: Training with engineered reward shaping
3. **Extension**: Hyperparameter tuning (learning rate)

## Environment: MountainCar-v0

A car is placed in a valley between two hills. The goal is to drive up the hill on the right to reach the flag. The car's engine is not strong enough to climb directly - it must build momentum by swinging back and forth.

**Observation Space:**
- Position: [-1.2, 0.6] (goal at >= 0.5)
- Velocity: [-0.07, 0.07]

**Action Space:** Discrete(3)
- 0: Push left
- 1: No push
- 2: Push right

**Default Reward:** -1 for each time step (very sparse)

## Configuration

All configurations are in YAML files under `config/`:

- `environment`: Gymnasium environment ID
- `algorithm`: DRL algorithm (DQN, PPO, A2C)
- `timesteps`: Total training timesteps
- `checkpoint_freq`: Save checkpoint every N steps
- `num_trials`: Number of training runs for robustness

### Custom Reward Parameters (config_custom.yaml)
- `position_reward_scale`: Reward weight for position progress toward goal
- `velocity_reward_scale`: Reward weight for building momentum
- `goal_bonus`: Bonus reward for reaching the goal
- `height_reward_scale`: Reward weight for potential energy (height)

## Custom Reward Design

### Problem Analysis
The default MountainCar reward (-1 per step) is extremely sparse. The agent receives no positive feedback until it reaches the goal, making exploration difficult. The car cannot climb directly to the goal - it must swing back and forth to build momentum.

### Custom Reward Function
Our custom reward addresses the sparse reward problem by providing intermediate feedback:

1. **Position reward**: Encourages moving toward the goal (right side)
2. **Velocity reward**: Rewards building momentum in either direction
3. **Directional bonus**: Extra reward for rightward velocity when on the right side
4. **Height reward**: Based on potential energy (terrain height = sin(3*position))
5. **Goal bonus**: Large bonus for reaching the goal

### Rationale
- The car starts in a valley and cannot climb directly
- It must swing left first to gain momentum, then swing right
- Rewarding velocity helps discover the swinging strategy
- Height reward provides a smoother gradient toward the goal
